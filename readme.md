# 给ai的命令
帮我做一个html单文件网页游戏，类似2048，但有几点区别。
1、合成条件不同。只有不同的数可以合成。为了避免某一次滑动有多种合成方式，当向某个方向滑动时，优先考虑这个方向上最远的相邻的两个数的合成，合成出来的数为原来两数之和，每一次滑动中先合成出来的数不能再参与这一次滑动中其它数的合成。
2、加入关卡机制。第i关的目标是合成2^(i+3)，合成2^(i+3)后清除所有大于等于2^(i+3)的数。(1<=i<=8)通过所有关卡即胜利。

# 对每个ai生成结果的概括：

## 元宝-deepseek(深度思考)
界面不错，但无法操控游戏。

## kimi(K2+长思考模式)
界面的外框有点歪，能玩，有小bug(不能向下移动)，但基本能玩。

## 豆包(深度思考)
界面挺好看，但是数没有正确显示。

## deepseek网页版(深度思考)
界面很华丽。数移动时的弹性视觉效果也很好。基本完成了所有功能！(唯一的问题就是过关时不会清除任何数)。

## chatgpt(免费用户+思考)
实现了所有功能！(不过显示数的格子有略微超出边界，不影响游玩)。

## grok(免费+专家)
界面美观，但功能上有点问题：  
1）每一次滑动都会清除所有大于本关目标的数
2）不会要求的顺序合成数

3）当同一行或同一列的数中间有空位时，滑动屏幕不会使它们合成，仅仅往滑动的方向移动并挤占空位
